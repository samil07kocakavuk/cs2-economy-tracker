# Product Direction

## Core Positioning

This product should not try to become a broad CS2 market directory like `cs2economy.com`.
Instead, it should focus on being an `inventory-first decision platform`.

Main value:
- analyze a user's inventory
- estimate value and post-fee Steam balance
- help decide what to sell and how to liquidate faster
- track collection goals
- evaluate sticker and craft premium over time

## Best Differentiators

### 1. Inventory Analyzer
- inventory value
- selected item value
- Steam fee adjusted value
- grouped duplicate items
- sticker and charm visibility
- actionable item-level review

### 2. Liquidation Planner
- fastest path to cash
- best path to maximum return
- balanced liquidation strategy
- per-item sell recommendations
- Steam balance versus cash-out guidance
- future route ideas such as buy here / sell there workflows

### 3. Collection Tracker
- collection pages
- collection completion cost
- missing items
- quality-based completion modes
- include knives connected to the collection concept
- collection investment angle

### 4. Sticker / Overpay Engine
- applied sticker parsing
- duplicate sticker handling
- craft popularity
- sticker market value context
- estimated premium bands
- confidence score

### 5. AI Advisor
- user can discuss buying, selling, and investment ideas
- should eventually use tools and live inventory data
- should not be only a plain LLM prompt layer
- best added after core calculators are reliable

## Recommended Product Scope

### Phase 1
- Inventory Analyzer
- Sell / Liquidation Planner
- Collection Tracker

### Phase 2
- Sticker overpay system
- craft analysis
- market comparison improvements

### Phase 3
- AI advisor with tool-backed answers

## Product Summary

Suggested positioning:

`Analyze your CS2 inventory, track collection goals, and plan the fastest or smartest way to turn skins into value.`

## Why This Direction Works

`cs2economy.com` is strong as a wide market and catalog data site.
This project can win by being more personal, more actionable, and more decision-oriented.

Main difference:

`Not a general market database, but a decision engine for the user's own inventory.`
